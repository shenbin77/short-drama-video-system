## API 网关 — one-api (localhost:3002) ✅

> 统一管理多供应商 API Key，负载均衡 + 自动切换，所有文本类 AI 调用走此网关。

### 渠道配置

| 供应商 | 渠道数 | 类型 | 模型 | 用途 |
|--------|--------|------|------|------|
| **火山引擎** | 3 (Lin/Shen/Mom) | type=40 | doubao-1-5-pro-32k, seed-2-0-pro, seed-1-6-flash, seedream-5-0 | 文本/对话 (国内直连,免费配额) |
| **Gemini** | 3 (3个Google账号) | type=24 | gemini-2.5-flash, flash-image, 3-pro-image-preview | 文本/图片 (需代理) |
| **DeepSeek** | 1 | type=1 | deepseek-chat, deepseek-reasoner | 文本/推理 (国内直连) |
| **Ollama** | 1 | type=11 | qwen2.5:3b 等本地模型 | 备用离线 |

```powershell

$env:HTTPS_PROXY="http://127.0.0.1:7890"
$env:NO_PROXY="localhost,127.0.0.1,volces.com,deepseek.com"
E:\one-api\one-api.exe --port 3002
# 账号: root / shenbin77
# 管理面板: http://localhost:3002
```

**## 双业务线总览

本项目包含 **两条独立业务线**，共享 Layer 1-6 基础设施，仅在内容来源和运营方式上不同。

```text
╔══════════════════════════════════════════════════════════════════════╗
║                                                                      ║
║    🏢 公司线 (B2B代工)              🚀 个人线 (全自动矩阵)           ║
║                                                                      ║
║    同事给小说章节                    Layer 0 AI自动写小说             ║
║         │                                │                           ║
║         │ (手动/Hermes导入)              │ (24h无人值守)             ║
║         ▼                                ▼                           ║
║    ┌──────────────────────────────────────────────┐                  ║
║    │           Layer 1-6  共享基础设施             │                  ║
║    │  内容源 → AI引擎 → 流水线 → 质控 → 后期 → 分发│                  ║
║    └──────────────────────────────────────────────┘                  ║
║         │                                │                           ║
║         ▼                                ▼                           ║
║    发给同事/上传红果              Layer 7 自动运营中心               ║
║    (人工交付)                     监控+调度+自进化+Hermes Agent监督  ║
║                                                                      ║
╚══════════════════════════════════════════════════════════════════════╝**
```

### 🏢 公司线 — B2B代工模式

| 环节 | 说明 |
|------|------|
| **输入** | 同事发来小说章节文本（10几章或整本） |
| **导入** | 通过 Hermes Agent 等 AI 助手输入到 ToonFlow |
| **生成** | Layer 1-5 全自动（大纲→分镜→视频→后期） |
| **交付** | 下载视频发给同事，同事上传到红果/各平台 |
| **特点** | 半自动，人在回路，按需执行，质量可控 |

### 🚀 个人线 — 全自动矩阵模式

| 环节 | 说明 |
|------|------|
| **输入** | Layer 0 AI 小说生成器自动写小说 |
| **设定** | 只需给定题材框架（如"仙侠+男频+升级流"） |
| **生成** | Layer 0→Layer 5 全链路无人值守 |
| **分发** | Layer 6 自动发布9平台（抖音/快手/小红书/红果/B站/YouTube/TikTok...） |
| **运营** | Layer 7 监控中心 + 数据反馈 + 自进化 + Hermes Agent 全程监督 |
| **特点** | 24h 运转，全托管，自我优化 |

### 双管线调度器 (小说按需 + 视频 24h)

小说生成很快 (~3-4h/本), 视频生成很慢 (~75h/本). 因此**小说按需生成, 视频 24h 不间断**.

```bash
# 启动 24h 视频管线 (自动按需生成小说)
python src/scheduler.py

# 只生成小说 (不做视频)
python src/scheduler.py --novel-only

# 先生成2个种子再开始
python src/scheduler.py --generate-seeds 2

# 只处理一章视频后退出
python src/scheduler.py --once

# 查看双管线状态
python src/scheduler.py --status
```

**管线A (小说)**: 视频队列<50章时自动触发, 种子池→小说生成→章节入视频队列→归档种子
**管线B (视频)**: 24h不间断, 从视频队列取章节→生成视频→分发→下一章

### AutoDream — 反思整理 (借鉴 KAIROS)

借鉴 Claude Code 的 KAIROS AutoDream 机制，系统在空闲时执行"反思性回顾"：

```text
触发时机: 调度器空闲(视频队列为空) / 每天凌晨 / 手动 dream 命令
    │
    ▼

    ▼

    ▼
Phase 3  检测跨维度模式 — 完播率衰减曲线/题材差距/播放量趋势
    ▼
Phase 4  修剪过时数据 — 清理旧日志/轮转梦境日志(保留最近30个)
    ▼
Phase 5  写入结构化梦境日志 → ops_reports/dreams/dream_YYYYMMDD_HHMM.md
```

### 母体模板自动进化 (template_evolution.py)

母体模板不再是静态文件。系统基于真实播放数据自动优化创作方法论：

```text
ops.db (完播率/互动率/流失点)
    │
    ▼
TemplateAnalyzer — 4维分析
    │ 节奏分析(前/中/后段完播率对比)
    │ 钩子分析(最佳/最差集的结尾特征)
    │ 流失点分析(完播率暴跌>20%的位置)
    │ 爆款特征分析(高分章节的共同模式)
    ▼
生成修订建议 → LLM(DeepSeek/Qwen) 生成新版模板
    │              保持6层方法论框架
    │              只修改数据支撑的条目
    │              每条标注 [数据驱动]
    ▼
自动质量门禁（无需人工审核）:
    │ ✓ 6层方法论结构完整
    │ ✓ 新模板长度≥旧版60%（防截断）
    │ ✓ 包含[数据驱动]标注（证明有数据支撑）
    ▼
版本化: templates/mother_template_v{N}.txt
    │
    ▼
效果验证: 新版小说完播率 vs 旧版
    ├─ 提升 → 保持新版（内容越来越好）
    └─ 下降>20% → 自动回滚到上一版
```

```bash
# 查看模板版本和进化历史
python src/template_evolution.py status

# 分析表现数据 → 生成优化建议
python src/template_evolution.py analyze

# 执行进化(需人工确认)
python src/template_evolution.py evolve

# 自动审批模式(用于 full 闭环)
python src/template_evolution.py evolve --auto-approve

# 回滚到上一版
python src/template_evolution.py rollback

# 比较两个版本差异
python src/template_evolution.py diff 2 3

# 自动回滚检查(新版效果差时触发)
python src/template_evolution.py check
```

## 角色一致性保障

每部小说的角色通过 ToonFlow 角色资产配置，系统自动加载并应用以下 6 层保障机制：

```text
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  CHARACTER_IDENTITY_CARDS (身份卡硬约束)                         │
│                                                                 │
│  每个角色包含：                                                │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │ age(年龄) | body(体型) | face(脸型) | hair(发型)   │    │
│  │ clothing(服装) | props(道具) | ban_words(禁用词) │    │
│  └─────────────────────────────────────────────────────────┘    │
│                                                                 │
│  示例（龙象焚天）:                                              │
│  • 叶辰: 青年/瘦削挺拔/剑眉星目/黑发高束/白底红金长袍/青岚剑  │
│  • 玄尘: 中年/微胖国字脸/浓眉鹰眼/紫金冠三须/深紫宗主袍  │
│  • 林婉儿: 青年/柔美纤细/柳叶眉杏眼/乌黑长发/浅蓝裙装   │
│                                                                 │
│  保障机制 (适用于所有小说):                                  │
│  1. 身份卡 → 每个prompt强制注入年龄/体型/脸型/发型/服装         │
│  2. 禁用词表 → ban_words阻止角色特征交叉污染                    │
│  3. sanitize_storyboard_prompt → 替换禁用服装词                 │
│  4. 角色参考图 → ComfyUI Klein+ReferenceLatent 参考图注入 (一致性V2 LoRA, 本地磁盘fallback)                  │
│  5. 剧情事实卡 → must_have / must_not_have 约束                 │
│  6. 固定prompt → 关键镜头完全绕过AI, 手写prompt直出              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 新小说如何配置角色

只需修改 **1个文件** `characters.json`（无需改动任何 Python 代码）：

1. 修改 `_meta` 字段：`novel`(小说名)、`genre`(题材)、`director_role`(AI导演角色)、`synopsis`(一句话简介)
2. 替换 `characters` 中的所有角色身份卡（外貌/服装/道具/禁用词/期望色系等）
3. 更新 `names_ban_in_prompt` 和 `banned_clothing` 列表
4. 在 ToonFlow UI 中创建新项目，添加角色资产（名称 + 参考图）
5. 将小说原文放到 `data/{小说名}/chXX.txt`
6. Pipeline 启动时自动从 `characters.json` + ToonFlow `getAssets` API 加载角色数据

## API 端点速查

> 说明：本节保留为历史接口速查表，仍有参考价值。
>
> 但具体 API 行为、当前调用路径、适配层约束与真实实现，请优先核对 `src/` 实际代码以及 `clusters/shared/adapters/`，不要把本节直接当作今天的唯一接口合同。

| `/other/login` | 登录获取token | `{"username":"admin","password":"admin123"}` |

| `/outline/addOutline` | 创建/更新大纲 | data字段为JSON字符串 |
| `/script/generateScriptApi` | 生成剧本内容 | 需已有script DB记录 |
| `/script/geScriptApi` | 获取剧本列表 | |
| `/script/generateScriptSave` | 保存剧本 | |
| `/assets/getAssets` | 获取资产列表 | `type="角色"` / `type="分镜"` |
| `/assets/generateAssets` | 生成分镜图/角色图 | id=0新建 |
| `/storyboard/generateShotImage` | 生成单张分镜图 | 返回Buffer字节 |
| `/storyboard/keepStoryboard` | 确认分镜 | 需完整字段集 |
| `/video/addVideoConfig` | 创建视频配置 | |

| `/video/getVideo` | 查询视频状态 | state: 0=生成中, 1=成功 |

## 已知限制

| 限制 | 说明 |
|------|------|
| **Klein 远景执行力一般** | 远景/大场面需精心调 prompt 引导构图 |
| **视频生成耗时** | LTX Desktop 1280×704 每段~90s，一集约30min |
| **API 429限流** | 高并发时触发，pipeline已内置指数退避重试 |
| **doubao 日限额** | 超出返回HTTP 400 |
| **LTX不支持并发** | 必须等idle状态再提交下一个请求 |
| **LTX vram_limit** | 必须设16(不是0)，0会3090满载卡死 |
| **LTX ALL_PROXY** | 环境变量会导致本地API 502，已用trust_env=False绕过 |
| **LTX多关键帧bug** | keyframeTimes帧数计算会除8导致关键帧挤在一起，大雄尚未修复 |
| **分镜图耗时** | 每张约25秒（Klein 8步, 热启动） |
| **VRAM泄漏** | 连续生成后VRAM不释放，已通过ComfyUI /free API修复 |

## 常见问题排查

### 1. API 429 限流

```
错误: HTTP 429 Too Many Requests
```

**原因**: 火山引擎 doubao 模型调用频率过高。

**解决**: Pipeline 已内置指数退避重试（10s → 30s → 60s，最多3次）。如果仍然频繁 429：
- 降低并发：在两次调用间增加 `time.sleep(2)`
- 检查火山引擎控制台是否有 QPS 限制
- 高峰期避开使用

### 2. doubao 日限额 400

```
错误: HTTP 400 / SetLimitExceeded
```

**原因**: 账号达到每日推理限额。

**解决**:
- 火山引擎控制台 → 模型推理 → 关闭"安全体验模式"
- 或申请提升限额
- 等待第二天限额重置

### 3. ToonFlow UI 崩溃 / 获取剧本列表失败

```
错误: ToonFlow UI 白屏或报错"获取剧本列表失败"
```

**原因**: `t_script` 表中有脏数据（字段缺失或重复记录）。

**解决**:
```bash
# 用 DB Browser for SQLite 打开
%APPDATA%\toonflow-app\db.sqlite

# 检查 t_script 表，删除异常记录
# pipeline 已有防重复机制，正常使用不会出现
```

### 4. 分镜图角色不一致 / 参考图丢失

```
日志: ❌ 参考图未使用: 该镜头走了failed(无角色参考图)
```

**原因**: ComfyUI 生成失败或超时，fallback 路径不带参考图。

**解决**:
- 检查 `quality_logs/ref_tracking_<script_id>.json` 确认哪些镜头走了 fallback
- 等限流缓解后重跑：`python src/batch_pipeline.py --from-step 6 --to-step 6 ...`

### 5. 视频生成失败 500

```
错误: generateVideo失败: Internal Server Error
```

**原因**: `generateVideo` 缺少 `configId` 字段，或 `t_videoConfig` 记录异常。

**解决**:
```bash
# 检查视频配置是否存在
python -c "
import sqlite3, os
conn = sqlite3.connect(os.path.join(os.environ['APPDATA'], 'toonflow-app', 'db.sqlite'))
rows = conn.execute('SELECT id, scriptId, prompt FROM t_videoConfig ORDER BY id DESC LIMIT 5').fetchall()
for r in rows: print(r)
conn.close()
"
```

### 6. 远景镜头被压成人物海报

**现象**: prompt 写了"大远景俯拍"，但生成的图片仍然是人物近景主体海报。

**原因**: FLUX 写实模型也倾向生成人物特写，需用远景描述词 (wide shot, scenery, from above, establishing shot) 引导构图。

**当前缓解措施**:
- 极简构图优先固定 prompt（减少外貌词，强调空间）
- 多候选择优（远景镜头自动增加候选数到6张）
- `_candidate_score` 对远景镜头的裁切和类型不符加重惩罚

**根本解决**: 已切换 Klein 9B + ReferenceLatent，构图控制力优于SDXL/majicFlus，但远景仍需精心调prompt。

### 7. FFmpeg 相关报错

```
错误: FileNotFoundError: ffmpeg / ffprobe not found
```

**解决**:
```bash
# Windows
winget install ffmpeg
# 或手动下载: https://ffmpeg.org/download.html
# 确保 ffmpeg.exe 和 ffprobe.exe 在 PATH 中
ffmpeg -version
```
