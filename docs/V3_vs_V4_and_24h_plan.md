# DeepSeek V3 vs V4 对比 & 24小时产线规划

## 一、DeepSeek V3 vs V4 综合对比

### 1.1 模型可用性 (2026-04-27 实测)

| 模型名 | API 可用 | 实际映射 | 说明 |
|--------|---------|---------|------|
| `deepseek-chat` | ✅ | deepseek-v4-flash | V3 已下线，自动升级到 V4 |
| `deepseek-reasoner` | ✅ | deepseek-v4-flash (带推理) | R1 也已替换为 V4 |
| `deepseek-v4-flash` | ✅ | deepseek-v4-flash | 直接调用 |
| `deepseek-v4-pro` | ✅ (列表中) | deepseek-v4-pro | 更强但更贵 |
| `deepseek-v3` | ❌ | 不存在 | **V3 已完全下线** |

**结论：没有 V3 vs V4 的选择问题 — V3 已不存在，只有 V4-Flash 和 V4-Pro。**

> 换算人民币 (1 USD ≈ 7.2 CNY):
> - V4-Flash: 输入 ¥1/M tok | 输出 ¥2/M tok
> - V4-Pro: 输入 ¥4/M tok | 输出 ¥8/M tok (折后 ¥3/¥6)

### 1.3 写小说成本估算

每章 ~2500字 ≈ ~3500 tokens (中文约1.4tok/字)
- system prompt: ~700 tokens (固定)
- 输出: ~3500 tokens

| 模型 | 每章成本 | 100章成本 | 速度 |
|------|---------|----------|------|
| **V4-Flash** | ¥0.008 | **¥0.8** | ~25s/章 |
| **V4-Pro** | ¥0.032 | ¥3.2 | ~40s/章 |

**结论：V4-Flash 写100章小说不到1元钱，完全没必要用 V4-Pro。**

## 三、项目拆分 & 24小时最优产线

### 3.1 项目拆分方案

将当前 `E:\视频项目` 的单体项目拆成 **4 个独立服务**，各自可独立启动、监控、重启：

```
E:\视频项目\
├── services/
│   ├── novel-factory/          # 📖 小说工厂 (纯API调用，不占GPU)
│   │   ├── novel_factory.py    # 主脚本: 自动批量写章节
│   │   ├── config.json         # 小说配置(角色/金手指/大纲)
│   │   ├── queue/              # 待处理队列
│   │   └── output/             # 成品章节
│   │
│   ├── storyboard-factory/     # 🎨 分镜工厂 (API调用，不占GPU)
│   │   ├── storyboard_gen.py   # 主脚本: 章节→大纲→分镜图
│   │   ├── queue/              # 待处理章节队列
│   │   └── output/             # 成品分镜(图+prompt)
│   │
│   ├── video-factory/          # 🎬 视频工厂 (Dreamina浏览器自动化)
│   │   ├── video_gen.py        # 主脚本: 分镜→Seedance视频
│   │   ├── queue/              # 待处理分镜队列
│   │   └── output/             # 成品视频片段
│   │
│   └── assembly-factory/       # 🎥 合成工厂 (FFmpeg，占CPU)
│       ├── assembly.py         # 主脚本: 片段→配音→字幕→成品
│       ├── queue/              # 待合成队列
│       └── output/             # 成品集视频
```

### 3.2 资源分配 — 并行不冲突

```
┌──────────────────────────────────────────────────────┐
│           24小时产线资源分配                           │
├──────────────┬───────────┬───────────┬────────────────┤
│ 服务          │ CPU      │ GPU/VRAM  │ 网络           │
├──────────────┼───────────┼───────────┼────────────────┤
│ 📖 小说工厂   │ 极低      │ 无        │ DeepSeek API   │
│ 🎨 分镜工厂   │ 低        │ 无        │ GPT Image2 API │
│ 🎬 视频工厂   │ 中(浏览器) │ 无        │ Dreamina 网站   │
│ 🎥 合成工厂   │ 高(FFmpeg)│ 无*       │ edge-TTS API   │
├──────────────┴───────────┴───────────┴────────────────┤
│ * GPU 24GB 当前空闲！未来可跑本地TTS (Qwen3-TTS)       │
└──────────────────────────────────────────────────────┘
```

**关键点：4个工厂全部走云端API，不占GPU。GPU 24GB 完全空闲，留给 TTS 或本地视频后备。**

### 3.3 流水线时序图 (队列驱动)

### 3.4 每日产能估算

| 环节 | 单次耗时 | 每集需要 | 瓶颈分析 |
|------|---------|---------|---------|
| **写小说** | 40s/章 | 1章/集 | ❌ 不是瓶颈 (全天能写2160章) |
| **分镜图** | 30-60s/张 (GPT Image2) | 7张/集 | ⚠️ 中等 (~7min/集) |
| **视频** | 3-5min/段 (Seedance 15s) | 6段/集 | **🔴 主瓶颈** (~24min/集) |
| **合成** | 2min/集 (FFmpeg+TTS) | 1次/集 | ❌ 不是瓶颈 |

**瓶颈 = Seedance 视频生成**

单账号: 24min/集 × 24h = **60集/天**
但 Dreamina 有每日限额 → 多账号轮换:
- 208个账号 × 每账号5次免费 = **1040次/天**
- 每集6段 → 1040÷6 = **~173集/天** (理论上限)
- 考虑失败率20% → **~138集/天**

**实际预估: 保守 60-80 集/天, 乐观 100+ 集/天**

## 四、决策总结

1. **V3 已不存在** → 无需对比，直接用 V4-Flash (reasoner模式写小说)
2. **成本极低** → 100章不到1元，没有成本顾虑
3. **拆4个独立工厂** → 小说/分镜/视频/合成，队列驱动，互不阻塞
4. **GPU 24GB 空闲** → 留给后续本地TTS (Qwen3-TTS)
5. **瓶颈在 Seedance** → 208账号轮换解决
6. **保守日产 60-80 集** → 够用
